# Technical Course Review Guidelines

## Purpose & Scope
These guidelines define a structured process for reviewing **online technical tutorials** (programming, cloud, AI/ML, data engineering, security, etc.). They ensure consistency, accuracy, and high instructional quality across the repository.

## Course Taxonomy
- **Technical**
  - Programming (Python, JavaScript, Go, Rust, …)
  - Cloud & DevOps (AWS, Azure, GCP, Terraform, Docker, …)
  - AI & ML (Generative AI, Prompt Engineering, Model Deployment, …)
  - Data Engineering (ETL, Spark, Snowflake, …)
  - Security & Compliance (OWASP, Cloud Security, …)
- **Non‑Technical** (Design, Soft Skills, …) – listed for completeness.

## Review Workflow
1. **Pre‑review checklist** – verify repository integrity, CI status, and required documentation.
2. **Content analysis** – evaluate each module against the criteria below.
3. **Feedback generation** – use the standard feedback template.
4. **Approval / Rejection** – assign a final score and decide on publishing.

## Evaluation Criteria (5‑point scale)
| Criterion | Description |
|-----------|-------------|
| **Accuracy** | Factual correctness, up‑to‑date references. |
| **Learning Objectives** | Clear, measurable outcomes for each lesson. |
| **Hands‑on Labs** | Working code samples, reproducible steps. |
| **Code Quality** | Style, linting, security best practices. |
| **Security Considerations** | Threat modeling, data privacy, safe defaults. |
| **Accessibility** | Captions, alt‑text, readable fonts, color contrast. |
| **Instructional Design** | Logical flow, pacing, engagement. |

## Scoring Rubric
- **5 – Excellent**: Exceeds expectations; no revisions needed.
- **4 – Good**: Minor improvements optional.
- **3 – Satisfactory**: Acceptable but requires several actionable suggestions.
- **2 – Poor**: Significant issues; must be addressed before publishing.
- **1 – Unacceptable**: Fundamental flaws; reject until reworked.
